Iterative Design: Designing an interactive interface for Pursuit

BACKGROUND

In this project, we designed an interactive interface using Figma for a startup called Pursuit. To design an interactive interface, we went through the full process of mocking up a solution to the startup's concept including sketching ideas of the interface, creating an interactive, high-fidelity prototype, conducting user testing on a final, revising prototype, and contacting the start up. These processes can be seen below.

Sketching and Wireframing

Sketching

We first started off by sitting down for 10 minutes and each developing our own set of sketches for how we imagine the app to look like. In these sketches, we designed interfaces such as the home screen, browsing for experiences or bookings, a screen for an individual experience or booking, help page, etc. in different design settings. It was very interesting to see the variety of design ideas our group came up with!

User Testing & Analysis of Results

Creating the User Test: With our newly redesigned mockup, we then tasked three test agents with using the Pursuit prototype to browse available guided outdoor experiences before booking a specific experience.

Test agents were given the following context before they began using the prototpe:

Pursuit is a marketplace for finding and booking outdoor lodging and guided experiences. Imagine you are looking to book a guided outdoor experience using the Pursuit mobile app.
Note that you will be interacting with a mockup made through prototyping software, rather than an actual mobile app. Not all buttons are clickable and input fields are not interactive. However, everything you need to complete the specified task has been built and can be reached by navigating through the app.
Please think out loud as you perform the specified task.

Then, test agents were given the following tasks:

  1. From the landing page, in your own words, what do you think you can do using this app?
  2. Browse the available guided outdoor experiences.
  3. Book the Catskills mountain hike.
  4. After completing the booking, review the details of your Catskills mountain hike booking.
  5. Try navigating back to the home page at some point while performing the tasks.

For tasks 2-5, test agents were prompted to answer whether they were succesful at the task (Yes/No) and to rate the difficulty of the task (on a five-point scale from very difficult to very easy).

They were then asked the following questions:

  • How easy or difficult was it for you to complete the specified tasks?
  • What were points of confusion for you while using the app, if any?
  • Describe your experience navigating between pages on the app.
  • Please feel free to share any other feedback you have regarding the app, including likes and dislikes.

Results & Analsysis: We received the results of our user testing, which indicated that test agents were able to succesfully complete all tasks but also highlighted areas for improvement.

Videos of agents completing our test can be accessed at the following links: User 1, User 2, User 3. GIFs of selected portions of the user testing are included below:

From reviewing the user testing sessions, we obtained the following takeaways:

  • All users successfully completed the overarching task of browsing outdoor experiences and then booking the Catskills mountain hike with ease. Referring to the numbered tasks above, the success rate for 2 and 3 was 100% with all testers rating the task as "Very Easy".
  • Users understood the back arrow and home button navigation well (with 100% success on task 5 and all testers rating it "Very Easy").
  • Each tester misunderstood the bottom navigation bar, with general confusion on the meanings of the icons. For example, one user interpreted the tent icon as a button for booking experiences, when in reality, it is for booking lodging.
  • While working on task 4, one user was frustrated that there was no page for reviewing already-booked lodging and experiences.
  • Testers appreciated the visual design of the app, particularly the large photos, but some desired additional photos for each experience.

Next Steps: We were happy to see user success on the given tasks! We were surprised by the confusion on the bottom navigation bar but grateful to have identified areas for improvement from the user testing. If we were to iterate on our designs once more, we would:

  • Choose more clear icons for the bottom navigation bar as well as include text describing what each icon navigates to
  • Add in a page for viewing already-booked lodging and experiences
  • Add more photos for each experience

Summary & Takeaways

In this project, we went through a full design process and iterated on our design based on various feedback. During this process, we learned:

  • When working on designs in groups, team members may have different opinions on design choices. It is important to ensure that all team members are on the same page when working on the design.
  • User testing and feedback is important! Through critiques and user testing, we discovered places our design could be improved that we had not noticed before.
  • Designing for mobile applications requires greater consideration of things such as accessibility as well as larger font and button tap sizes to improve usability.
